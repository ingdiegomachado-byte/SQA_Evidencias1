# Evidencias SQA - Extensión de Captura de Pantalla

**Versión:** 3.7.0  
**Descripción:** Herramienta profesional de captura de pantalla diseñada para analistas de QA, que permite generar evidencias con metadatos integrados, gestión de historial, edición avanzada y adjuntos desde el portapapeles/arrastre.

---

## 📋 Changelog

### v3.7.0 — 2026-05-02
**Feature: Adjuntos desde Clipboard y Drag & Drop**
- **Clipboard Paste**: Permite pegar imágenes directamente en el visor SQA (`Ctrl+V`). Las imágenes se procesan, normalizan y guardan automáticamente en el historial.
- **Drag & Drop**: Soporte para arrastrar archivos de imagen desde el explorador de archivos directamente al visor para adjuntarlos como evidencia.
- **Encabezado Integrado**: Se unificó la lógica del encabezado SQA; ahora las imágenes externas pegadas o arrastradas incluyen automáticamente el encabezado institucional (Logo, Título, Fecha y Navegador).
- **Calidad y Normalización**: Las imágenes adjuntas se escalan a un ancho estándar de 627px usando suavizado de alta calidad para mantener la nitidez.
- **Arquitectura Modular**: Introducción de `clipboardHandler.js`, `imageProcessor.js` y `storageManager.js` para gestionar el nuevo flujo de adjuntos.

### v3.6.5 — 2026-04-23
**Perf: Reducción de latencia en captura (~60% más rápido)**

#### Optimizaciones de Tiempos
- **`prepareTime`**: Reducido de 800ms → 300ms (con elementos scrollables) y de 100ms → 50ms (sin elementos scrollables). Primer frame de captura llega mucho antes.
- **`waitTime` base**: Reducido de 500ms → 200ms por página (scrollTop > 0) y de 150ms → 80ms en la primera página.
- **`waitTime` extra dinámico**: Reducido de +800ms → +300ms cuando la página crece durante la captura.
- **`waitTime` extra en bloque final**: Reducido de +1000ms → +400ms.
- **`_waitTime` en verificación final**: Reducido de 1000ms → 300ms cuando `scrollTop > 0`, y de 100ms → 60ms en el caso base.
- **`isScrollLoadedElement` timeout**: Reducido de 1000ms → 400ms. Elimina el principal cuello de botella en páginas sin lazy-load.
- **`changStyleForShot` delay**: Reducido de 100ms → 60ms (por página) y de 20ms → 10ms (setup inicial).
- **Service Worker post-inyección**: Delay reducido de 500ms → 150ms al inyectar `content.js` por primera vez.
- **Service Worker retry delay**: Reducido de 300ms → 150ms entre reintentos de mensaje.

#### Impacto estimado
| Escenario | Antes | Después |
|:---|:---|:---|
| Página simple (1 pantalla) | ~650ms | ~130ms |
| Página con scroll (3 pantallas) | ~2.8s | ~1.0s |
| Primera carga (inyección) | ~800ms | ~300ms |

- **Renderizado Final**: Los nodos se fusionan al canvas al guardar para preservar las anotaciones en el PNG final.

### v3.6.0 — 2026-04-22
**Feature: Editor avanzado con objetos seleccionables, crop, blur, resaltador y flujo neutral**

#### Comportamiento al entrar en edición
- **Zoom 50% por defecto** al abrir el editor (antes 100%).
- **Estado neutro**: ninguna herramienta está activa al iniciar el editor. El usuario debe seleccionar explícitamente una herramienta.
- **Navegación libre**: cuando ninguna herramienta está activa, el scroll del mouse mueve el área de trabajo sin interferir con las herramientas.
- **Esc multi-función**: cierra el input de texto flotante → cancela crop → deselecciona objeto → regresa al estado neutro (en ese orden de prioridad).

#### Herramientas nuevas
- **🔲 Recortar (Crop):** Define un área arrastrando sobre el canvas. Los 4 handles de esquina permiten ajustar el recorte. Doble clic o Enter confirma el recorte; Esc cancela. El canvas se redimensiona al área recortada.
- **🌟 Resaltador (Highlighter):** Pincel semitransparente (`globalAlpha: 0.38`) que resalta áreas sin tapar el contenido original. Atajo: `H`.
- **🔵 Desenfoque (Blur):** Pixelación de la región seleccionada (downsample + upsample) para ocultar datos sensibles. Se aplica inmediatamente a la imagen base. Atajo: `B`.

#### Objetos seleccionables e interactivos
- **Herramienta Seleccionar** (`V`): Permite hacer clic sobre cualquier objeto dibujado (flecha, cuadro, texto, resaltador) para seleccionarlo.
- **Mover** con drag del mouse o con las **teclas de flecha** (1px por tecla; 10px con Shift).
- **Redimensionar** arrastrando los handles de esquina (cuadros, textos).
- **Eliminar** objeto seleccionado con `Delete` o `Backspace`.
- **Doble clic** en un texto seleccionado para re-editarlo.

#### Cuadros de texto inteligentes
- Dimensiones del cuadro ajustables via handles de resize en modo selección.
- El tamaño de fuente se **auto-escala** para que el texto siempre quepa dentro del ancho del cuadro definido.

#### UX y teclado
- **Cursores dinámicos** por herramienta y estado: `crosshair` (dibujo), `text` (texto), `default` (seleccionar), `grab/grabbing` (mover objeto), `move` (hover sobre objeto), `nwse-resize` (handle de resize).
- **Atajos de teclado de herramientas**: `V` Seleccionar, `A` Flecha, `R` Cuadro, `T` Texto, `H` Resaltador, `B` Blur, `C` Recortar.
- **Status bar** en el editor: muestra instrucción contextual y coordenadas del cursor en tiempo real.
- Toggle de herramienta: hacer clic en una herramienta activa la deselecciona (regresa a neutro).

#### Técnico
- Sistema de objetos vectoriales en memoria (`objects[]`) separado de la imagen base (`_baseImageData`). Los objetos se renderizan sobre la base en cada frame.
- `commitObjectsToBase()`: fusiona todos los objetos al `_baseImageData` antes de guardar o aplicar crop.
- `willReadFrequently: true` en el contexto 2D para evitar warnings de `getImageData`.
- **Archivos modificados:** `result.html`, `result.js`, `Evidencias_SQA.md`

## 🛠️ Arquitectura Técnica

### 1. Núcleo del Sistema
- **`manifest.json`**: Permisos (`activeTab`, `scripting`, `storage`, `clipboardWrite`), service-worker, recursos web accesibles.
- **`service-worker.js`**: Orquestador de capturas, gestión de mensajes, administración de IndexedDB (`SQAEvidence`).
- **`jsconfig.json`**: IntelliSense y validación de tipos.

### 2. Capa de Inyección (Content Script)
- **`content.js`** (2400+ líneas): Cálculo DOM, scroll automático, encabezado dinámico en Canvas, tratamiento de elementos fijos.

### 3. Interfaz de Usuario y Visor
- **`result.html`**: Tres vistas: Visor, Historial, Editor.
- **`result.js`**: Visor, historial, editor vectorial con objetos seleccionables, crop, blur, resaltador, undo/redo, zoom.

---

## 📊 Flujo de Datos

1. **Activación**: Click en el icono de la extensión.
2. **Inyección**: `service-worker` inyecta `content.js`.
3. **Procesamiento**: Capturas parciales unidas en Canvas con encabezado.
4. **Almacenamiento**: Imagen Base64 en IndexedDB.
5. **Visualización**: `result.html` muestra la evidencia.
6. **Edición opcional**: Editor vectorial permite anotar y guardar cambios.

## 🚀 Funcionalidades Clave

| Función | Descripción |
|:---|:---|
| Encabezado Automático | URL, fecha, navegador en cada captura |
| Historial Local | IndexedDB sin servidores externos |
| Editor Vectorial | Flechas, cuadros, textos, resaltador, blur, crop |
| Objetos Interactivos | Seleccionar, mover, redimensionar, eliminar |
| Undo/Redo | Historial de estados completo |
| Zoom Editor | Ctrl+Rueda, botones, pinch-to-zoom |
| Descarga ZIP | Exportar todas las evidencias comprimidas |

## 📁 Estructura de Archivos

| Archivo | Función |
|:---|:---|
| `manifest.json` | Descriptor de la extensión |
| `service-worker.js` | Lógica de fondo y base de datos |
| `content.js` | Lógica de captura y manipulación del DOM |
| `result.html` / `result.js` | Visor, historial y editor |
| `SQA.png` | Identidad visual (Logo) |
| `jszip.min.js` | Compresión ZIP para descarga masiva |

---

> **Nota para Desarrolladores:** El sistema de unión de imágenes (`splicingImagesAndSendAction`) soporta alturas de hasta 90,000px, gestionando los límites de memoria de navegadores Chromium. El editor v3.6.0 usa un sistema de doble capa: `_baseImageData` (imagen rasterizada) + `objects[]` (objetos vectoriales en memoria), que se fusionan al guardar o aplicar crop.
